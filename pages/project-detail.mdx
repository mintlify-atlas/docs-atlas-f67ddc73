# Project Detail Page

The project detail page (`src/app/projects/[slug]/page.tsx`) is a dynamically generated page that displays comprehensive information about individual projects using Next.js App Router's dynamic routing.

## Dynamic Routing

The page uses Next.js 13+ App Router dynamic route segments:

```
src/app/projects/[slug]/page.tsx
```

### File Structure:
```
src/app/
└── projects/
    ├── page.tsx              # Projects listing
    └── [slug]/
        └── page.tsx          # Dynamic project detail (this file)
```

## Static Site Generation (SSG)

The page uses `generateStaticParams` to pre-render all project pages at build time:

```tsx
export async function generateStaticParams() {
    const slugs = getAllSlugs();
    return slugs.map((slug) => ({ slug }));
}
```

### How It Works:
1. `getAllSlugs()` returns an array of all project slugs
2. Each slug is mapped to an object with `slug` property
3. Next.js generates a static page for each slug at build time

**Example output:**
```typescript
[
  { slug: "task-manager" },
  { slug: "ecommerce-api" },
  { slug: "portfolio-site" }
]
```

## Dynamic Metadata

Metadata is generated dynamically based on the project:

```tsx
export async function generateMetadata({ params }: Props): Promise<Metadata> {
    const { slug } = await params;
    const project = getProjectBySlug(slug);

    if (!project) {
        return {
            title: "Project not found",
        };
    }

    return {
        title: project.title,
        description: project.description,
    };
}
```

### Async Params

Note that `params` is awaited because Next.js 15+ uses async params:

```tsx
type Props = {
    params: Promise<{ slug: string }>;
};
```

## Page Component

```tsx
export default async function ProjectDetailPage({ params }: Props) {
    const { slug } = await params;
    const project = getProjectBySlug(slug);

    if (!project) {
        notFound();
    }

    // Conditional flags for different sections
    const hasExtendedInfo = project.problem || project.solution || project.result;
    const hasArchitecture = project.architecture && project.architecture.length > 0;
    const hasChallenges = project.challenges && project.challenges.length > 0;
    const hasLessons = project.lessonsLearned && project.lessonsLearned.length > 0;
    const isApiProject = project.displayType === "api";
    const hasCapabilities = project.capabilities && project.capabilities.length > 0;
    const hasEndpoints = project.apiPreview && project.apiPreview.endpoints.length > 0;
    const hasSampleRequest = project.sampleRequest;
    const hasImages = project.images && project.images.desktop.length > 0;

    return (
        // Page content
    );
}
```

## Page Structure

The page consists of multiple conditional sections:

1. **Back Link** - Navigation back to projects
2. **Hero Section** - Title and description
3. **Stats Row** - Status, role, project type badges
4. **Problem/Solution/Result** - Three-column overview
5. **Project Gallery** - Image carousel and mobile preview (for web apps)
6. **API Capabilities** - Feature badges (for API projects)
7. **API Endpoints** - Endpoints table (for API projects)
8. **Sample Request** - Code example (for API projects)
9. **Key Features** - List of highlights
10. **Tech Stack** - Grouped or flat technology list
11. **Architecture** - Patterns and practices
12. **Technical Challenges** - Problems and solutions
13. **Links** - Repository and live demo
14. **Lessons Learned** - Key takeaways

## Status Badge Component

Custom component for displaying project status:

```tsx
function StatusBadge({ status }: { status: string }) {
    const statusConfig = {
        completed: { label: "Completed", color: "text-emerald-400 bg-emerald-400/10 border-emerald-400/20" },
        "in-progress": { label: "In Progress", color: "text-amber-400 bg-amber-400/10 border-amber-400/20" },
        maintenance: { label: "Maintenance", color: "text-blue-400 bg-blue-400/10 border-blue-400/20" },
    };
    const config = statusConfig[status as keyof typeof statusConfig] || statusConfig.completed;
    return (
        <span className={`inline-flex items-center gap-1.5 px-3 py-1.5 text-sm font-medium border rounded-full ${config.color}`}>
            <span className="w-1.5 h-1.5 rounded-full bg-current" />
            {config.label}
        </span>
    );
}
```

                </div>

        </div>

### Layout Breakdown:
- **Desktop**: `lg:grid-cols-[1fr_280px]` creates 70/30 split
- **Mobile**: Stacks vertically
- **Carousel**: Takes flexible 1fr space
- **Phone frame**: Fixed 280px width
