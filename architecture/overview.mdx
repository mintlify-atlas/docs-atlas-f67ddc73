## Project Overview

The Mario Rodriguez Portfolio is a modern web application built with **Next.js 16** using the App Router architecture. It showcases professional projects with detailed case studies, supports both web application and API project displays, and includes a contact form with email integration.

## Technology Stack

### Frontend
- **Framework**: Next.js 16 with App Router
- **Language**: TypeScript
- **Styling**: Tailwind CSS with custom design system
- **Animations**: Motion library (motion/react)
- **Font**: Space Grotesk (Google Fonts)
- **Image Optimization**: Next.js Image component with AVIF/WebP support

### Backend & Services
- **API Routes**: Next.js Route Handlers (App Router)
- **Email Service**: Resend API
- **Analytics**: Vercel Speed Insights

### Hosting & Infrastructure
- **Platform**: Vercel
- **Image CDN**: Vercel Blob Storage
- **Domain**: mariorodriguez.dev

## Architecture Principles

### 1. **Server-First Rendering**
- Pages use React Server Components by default for optimal performance
- Client components (`"use client"`) only where interactivity is needed
- Static generation for project pages using `generateStaticParams()`

### 2. **Type Safety**
- Comprehensive TypeScript interfaces for all data structures
- Centralized type definitions in `src/data/projects.ts`
- Type-safe API routes with proper error handling

### 3. **Component Composition**
- Atomic, reusable components with single responsibilities
- Props-based configuration for flexibility
- Consistent styling patterns using Tailwind utility classes

### 4. **Data-Driven Content**
- All project data centralized in `projects.ts`
- Single source of truth for project information
- Easy to add/modify projects without touching component code

### 5. **Progressive Enhancement**
- Works without JavaScript for core content
- Enhanced with animations and interactivity when JS loads
- Semantic HTML structure for accessibility

## Key Features

### Dynamic Project System
- Supports two display types: `webapp` and `api`
- Conditional rendering based on project type
- Static generation of all project detail pages at build time

### Responsive Design
- Mobile-first approach with Tailwind breakpoints
- Adaptive layouts (70/30 split for desktop carousel + mobile frame)
- Collapsible mobile navigation

### Performance Optimizations
- Image optimization with Next.js Image (AVIF, WebP)
- Remote pattern allowlist for Vercel Blob storage
- Lazy loading for non-critical components
- Speed Insights monitoring

### Internationalization Ready
- Currently in Spanish (`lang="es"`)
- Structured for easy i18n implementation
- SEO metadata in appropriate language
