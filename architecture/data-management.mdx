## Overview

All portfolio project data is centralized in **`src/data/projects.ts`**. This file serves as the single source of truth for:

- Project metadata and content
- TypeScript type definitions
- Helper functions for data retrieval

This data-driven approach allows adding/editing projects without modifying components or pages.

---

## Type Definitions

### Core Types

#### ProjectImage

```typescript
export type ProjectImage = {
  src: string;     // URL or path to image
  alt: string;     // Accessibility description
};
```

**Usage**: Desktop screenshots, mobile screenshots

**Example**:
```typescript
{
  src: "https://9jaubfal4qpj0a4o.public.blob.vercel-storage.com/booking/hero.png",
  alt: "Booking platform landing page"
}
```

---

#### TechCategory

```typescript
export type TechCategory = {
  category: string;           // Category name (e.g., "Frontend", "Backend")
  technologies: string[];     // List of technologies
};
```

**Usage**: Grouped tech stack display in project details

**Example**:
```typescript
{
  category: "Frontend",
  technologies: ["Blazor Server", "MudBlazor", "MudBlazor.Extensions"]
}
```

---

#### Challenge

```typescript
export type Challenge = {
  title: string;      // Problem statement
  solution: string;   // How it was solved
};
```

**Usage**: Technical challenges section in project case studies

**Example**:
```typescript
{
  title: "State consistency between steps",
  solution: "Centralized state service (BookingStateService) to maintain booking data"
}
```

### API-Specific Types

#### HttpMethod

```typescript
export type HttpMethod = "GET" | "POST" | "PUT" | "DELETE" | "PATCH";
```

**Usage**: Type-safe HTTP method declarations

---

#### Endpoint

```typescript
export type Endpoint = {
  method: HttpMethod;   // HTTP method
  path: string;         // Endpoint path
  label?: string;       // Optional description
};
```

**Usage**: API endpoint definitions

**Example**:
```typescript
{
  method: "GET",
  path: "/items",
  label: "List inventory items"
}
```

---

#### ApiPreview

```typescript
export type ApiPreview = {
  baseUrl?: string;       // Base URL (e.g., "/v1")
  endpoints: Endpoint[];  // List of endpoints
};
```

**Usage**: Terminal-style API preview in project cards and detail pages

---

#### SampleRequest

```typescript
export type SampleRequest = {
  method: string;      // HTTP method
  endpoint: string;    // Full endpoint path
  headers?: string;    // Request headers (formatted)
  body?: string;       // Request body JSON
  response?: string;   // Response body JSON
};
```

**Usage**: Code block examples in API project details

**Example**:
```typescript
{
  method: "POST",
  endpoint: "/v1/items",
  headers: "Authorization: Bearer <token>\nContent-Type: application/json",
  body: `{
  "sku": "WIDGET-001",
  "name": "Industrial Widget",
  "costPrice": 15.50

  "id": "770e8400-e29b-41d4-a716-446655440002",
  "sku": "WIDGET-001",
  "isActive": true
}`
}
```

### Main Project Type

#### Project

```typescript
export type Project = {
  // REQUIRED FIELDS
  slug: string;               // URL-friendly identifier
  title: string;              // Project name
  description: string;        // Short description (1-2 sentences)
  stack: string[];            // Technology tags (simple list)
  highlights: string[];       // Key features/accomplishments

  // OPTIONAL MEDIA
  images?: {
    desktop: ProjectImage[];  // Desktop screenshots (first = preview)
    mobile?: ProjectImage;    // Single mobile screenshot
  };
  repo?: string;              // GitHub repository URL
  live?: string;              // Live demo URL

  // DISPLAY TYPE
  displayType?: "webapp" | "api";

  // EXTENDED METADATA
  status?: "completed" | "in-progress" | "maintenance";
  role?: string;              // Your role (e.g., "Full-Stack Developer")
  projectType?: string;       // Type (e.g., "Enterprise Web App")
  keyOutput?: string;         // Main deliverable

  // CASE STUDY SECTIONS
  problem?: string;           // Problem statement
  solution?: string;          // Solution approach
  result?: string;            // Outcome/impact

  // TECHNICAL DETAILS
  stackDetails?: TechCategory[];   // Grouped tech stack
  architecture?: string[];         // Architecture patterns
  challenges?: Challenge[];        // Technical challenges
  lessonsLearned?: string[];       // Key takeaways

  // API-SPECIFIC FIELDS
  apiPreview?: ApiPreview;         // Endpoint preview
  capabilities?: string[];         // API capabilities (badges)
  sampleRequest?: SampleRequest;   // Example request/response
};
```

## Helper Functions

### getProjectBySlug

```typescript
export function getProjectBySlug(slug: string): Project | undefined {
  return projects.find((project) => project.slug === slug);
}
```

**Usage**: Fetch project data in dynamic route

```typescript
const project = getProjectBySlug("booking-platform");
if (!project) notFound();
```

---

### getAllSlugs

```typescript
export function getAllSlugs(): string[] {
  return projects.map((project) => project.slug);
}
```

**Usage**: Generate static paths at build time

```typescript
export async function generateStaticParams() {
  const slugs = getAllSlugs();
  return slugs.map((slug) => ({ slug }));
}
```

**Output**:
```typescript
[
  { slug: "booking-platform" },
  { slug: "payroll-platform" },
  { slug: "inventory-api" }
]
```

## Adding a New Project

To add a new project to the portfolio:

### 1. Add project object to array

```typescript
export const projects: Project[] = [
  // ... existing projects
  {
    slug: "my-new-project",
    title: "My New Project",
    description: "A brief description",
    stack: ["Next.js", "TypeScript"],
    highlights: [
      "Feature 1",
      "Feature 2",
    ],
    images: {
      desktop: [
        { src: "/images/project.png", alt: "Screenshot" }
      ]
    },
  },
];
```

### 2. (Optional) Upload images to Vercel Blob

Images are hosted on Vercel Blob Storage:
```
https://9jaubfal4qpj0a4o.public.blob.vercel-storage.com/
```

### 3. Rebuild the site

```bash
npm run build
```

Next.js will automatically:
- Generate a new static page at `/projects/my-new-project`
- Update the projects listing page
- Include it in the homepage preview

## Type Safety Benefits

### 1. Autocomplete in IDEs

TypeScript provides IntelliSense for all project properties:

```typescript
const project = getProjectBySlug("booking-platform");
project?.  // IDE suggests: slug, title, description, stack, etc.
```

### 2. Compile-time Error Detection

```typescript
// ❌ Error: Type '"INVALID"' is not assignable to type 'HttpMethod'
{ method: "INVALID", path: "/test" }

// ✅ Valid
{ method: "GET", path: "/test" }
```

### 3. Refactoring Safety

Renaming a type property automatically updates all usages or shows errors.

## Best Practices

1. **Keep descriptions concise** (1-2 sentences max)
2. **Use action verbs** for highlights ("Built", "Implemented", "Designed")
3. **Order images logically** (hero shot first, then features)
4. **Group technologies** in `stackDetails` by logical categories
5. **Write challenges** as problem → solution pairs
6. **Make slugs URL-friendly** (lowercase, hyphens, no spaces)
7. **Host images externally** (Vercel Blob, Cloudinary, etc.)
