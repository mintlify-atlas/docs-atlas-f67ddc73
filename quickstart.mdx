# Quick Start Guide

Get the portfolio website running on your local machine in just a few steps. This guide covers installation, development, and building for production.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js**: Version 20 or higher
- **Package Manager**: npm, yarn, or pnpm
- **Git**: For cloning the repository

<Note>
  This project uses **pnpm** as the default package manager. The lockfile is `pnpm-lock.yaml`.
</Note>

## Installation

<Steps>
  <Step title="Clone the Repository">
    Clone the portfolio source code to your local machine:

    ```bash
    git clone https://github.com/mariorodriguez/portfolio.git
    cd portfolio
    ```
  </Step>

  <Step title="Install Dependencies">
    Install the required packages using your preferred package manager:

    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```

      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>

    This will install all dependencies from `package.json`:

    ```json title="package.json" {11-20}
    {
      "dependencies": {
        "@vercel/blob": "^2.0.1",
        "@vercel/speed-insights": "^1.3.1",
        "motion": "^12.29.2",
        "next": "16.1.6",
        "react": "19.2.4",
        "react-dom": "19.2.4",
        "react-icons": "^5.5.0",
        "resend": "^6.9.1"
      }
    }
    ```
  </Step>

  <Step title="Configure Environment (Optional)">
    For full functionality including contact form, create a `.env.local` file:

    ```bash
    touch .env.local
    ```

    Add your Resend API key for email functionality:

    ```bash title=".env.local"
    RESEND_API_KEY=your_resend_api_key_here
    ```

    <Warning>
      The site will run without environment variables, but the contact form won't send emails.
    </Warning>
  </Step>

  <Step title="Start Development Server">
    Run the development server:

    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```

      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>

    The development server will start at `http://localhost:3000`.

    <Check>
      Open [http://localhost:3000](http://localhost:3000) in your browser to see the portfolio.
    </Check>
  </Step>
</Steps>

## Available Scripts

The portfolio includes several npm scripts for development and production:

```json title="package.json" {5-10}
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "lint": "eslint"
  }
}
```

### Development

```bash
npm run dev
```

Starts the Next.js development server with:
- Hot module replacement
- Fast refresh for instant updates
- TypeScript type checking
- ESLint error reporting

### Production Build

```bash
npm run build
```

Creates an optimized production build:
- Minifies JavaScript and CSS
- Optimizes images (AVIF/WebP)
- Generates static pages
- Tree-shakes unused code

### Production Server

```bash
npm run start
```

Starts the production server after building. Requires `npm run build` first.

### Linting

```bash
npm run lint
```

Runs ESLint with Next.js configuration to check for code quality issues.

## Next.js Configuration

The site is configured for optimal image handling:

```typescript title="next.config.ts"
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  images: {
    formats: ["image/avif", "image/webp"],
    remotePatterns: [
      {
        protocol: "https",
        hostname: "9jaubfal4qpj0a4o.public.blob.vercel-storage.com",
        pathname: "/**",
      },
    ],
  },
};

export default nextConfig;
```

<Info>
  Images are served from Vercel Blob Storage and automatically optimized to AVIF/WebP formats.
</Info>

## TypeScript Configuration

The project uses strict TypeScript settings:

```json title="tsconfig.json" {7}
{
  "compilerOptions": {
    "target": "ES2017",
    "strict": true,
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

Use the `@/` alias for absolute imports:

```typescript
import Navbar from "@/components/Navbar";
import { projects } from "@/data/projects";
```

## Troubleshooting

### Port Already in Use

If port 3000 is occupied, Next.js will automatically use the next available port (3001, 3002, etc.).

### Module Not Found

Clear the cache and reinstall dependencies:

```bash
rm -rf node_modules .next
pnpm install
pnpm dev
```

### TypeScript Errors

Run the TypeScript compiler to see detailed errors:

```bash
npx tsc --noEmit
```

### Image Loading Issues

Ensure remote patterns are configured in `next.config.ts` for external images.
